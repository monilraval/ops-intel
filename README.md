# OpsIQ — Product Operations Intelligence Suite

> **Live Demo → [monilraval.github.io/ops-intel.html](https://monilraval.github.io/ops-intel.html)**

A world-first interactive portfolio project that lets a hiring manager **use** the tools I would build — not just read about them.

Built by **Monil Raval** — Product Owner & Operations Professional based in Kaufbeuren, Bavaria, Germany.

---

## What Is This?

OpsIQ is a fully interactive, browser-based Product Operations Intelligence Suite. Every module is grounded in real professional experience from Amazon EU (Operations) and AGCO GmbH (Product Owner, PIM & E-Commerce).

Most product management portfolios are static documents. This one runs live in your browser.

---

## What's Inside — 7 Interactive Modules

| Module | What It Does | Professional Origin |
|---|---|---|
| **Live Dashboard** | Real-time KPI simulation — UPH, defect rate, scan compliance, bottleneck heatmap, live alerts | Amazon EU — gaps I observed across 400+ shifts |
| **Shift Simulator** | Adjust day, shift, season, volume → get staffing recommendation + demand curve | ShiftSense AI hypothesis — predictive staffing model |
| **Bottleneck Finder** | A3 Lean analysis of 3 real process gaps I documented at Amazon | Direct observation — shift handover, reactive staffing, inbound-stow delays |
| **RICE Prioritiser** | Live feature scoring tool — input any feature, get RICE score instantly | AGCO — exact framework used to prioritise PIM backlog across 5 country teams |
| **Backlog Analyser** | Real sprint backlog structure from AGCO PIM project with user stories, epics, and — crucially — features deliberately NOT built with written rationale | AGCO — 127 stories across 12 months |
| **Sprint Roadmap** | 12-month delivery timeline including a mid-project scope change and how it was resolved | AGCO — real project milestone documentation |
| **Impact Timeline** | Chronological evidence trail of career decisions and outputs, honestly scoped | Full 4-year career in Germany |

## The Problem This Solves

Standard Product Owner portfolio: "I managed a backlog at Company X."

This portfolio: *opens OpsIQ, navigates to Backlog Analyser* — "Here is the exact sprint structure I used, here are the user stories I wrote, and here — most importantly — are the three features I deliberately chose NOT to build this quarter, and here is the written rationale I used to defend those decisions to five country stakeholders."

That conversation wins interviews.

## Technical Stack

- Pure HTML5 / CSS3 / Vanilla JavaScript — zero dependencies, zero frameworks
- Canvas API for real-time chart rendering
- Live KPI simulation with 4-second auto-refresh
- Fully responsive — works on desktop and mobile
- Single file — deploy anywhere by uploading one `.html` file

No build step. No npm. No server required.

## Why I Built This

I am currently an L1 Operations Associate at Amazon EU in Graben, Bavaria — deliberately building ground-up operational experience after 12 months as Product Owner at AGCO GmbH.

The single most common feedback I received from German recruiters: *"Your experience sounds interesting, but we can't see your work."*

OpsIQ is my answer to that. Every module in this tool represents something I lived, not something I read about.

The RICE prioritiser is the exact tool I used when five country managers each believed their feature request was the highest priority. The A3 bottleneck analysis covers problems I documented across real shifts. The backlog structure mirrors the sprint organisation I built at AGCO.
